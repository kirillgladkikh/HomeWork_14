# Проект "Домашнее задание 14"

## Описание:

Проект "Домашнее задание 17.1" - это код на Python покрытый тестами.
Работа над магазином.

Реализованы:
 - классы Product и Category. 
 - добавлены геттеры и сеттеры
 - добавлены Магические методы
 - созданы классы наследники: Smartphone, LawnGrass
 - создан абстрактный класс и миксин
 - обработка попытки создания товара с нулевым количеством
 - метод, который подсчитывает средний ценник всех товаров

### ДЗ 17.1

Инициализации экземпляра класса:
- При инициализации экземпляра класса Product происходит обработка попытки создания товара с нулевым количеством.
- Если пользователь попытается создать продукт с нулевым количеством, выбрасывается ошибка ValueError с сообщением «Товар с нулевым количеством не может быть добавлен».

Метод подсчета среднего ценника всех товаров:
- Реализован метод подсчета среднего ценника всех товаров в классе Category, который складывает только цены (self.price) всех товаров категории и делит полученную сумму на количество товаров в категории.
- В методе обработан случай, когда в категории нет товаров. В случае если такое происходит, возвращается ноль.

### ДЗ 16.2

Базовый класс:
- Создан базовый абстрактный класс с именем BaseProduct, который является родительским для классов продуктов.
- Классы «Смартфон» и «Трава газонная» наследуются только от класса Product.
- Выделена общая для каждого продукта функциональность и описана в абстрактном классе.

Класс-миксин
- Реализован класс-миксин, который при создании объекта распечатывает в консоль информацию о том, от какого класса и с какими параметрами был создан объект.
- Миксин добавлен в цепочку наследования класса Product.

### ДЗ 16.1

Классы-наследники:
- Созданы два класса наследников класса Product: «Смартфон» (Smartphone) и «Трава газонная» (LawnGrass).
- Класс «Смартфон» (Smartphone) расширен атрибутами: производительность (efficiency), модель (model), объем встроенной памяти (memory), цвет (color).
- Класс «Трава газонная» (LawnGrass) расширен атрибутами: страна-производитель (country), срок прорастания (germination_period), цвет (color).

Ограничения сложения:
- Доработана функциональность сложения таким образом, чтобы можно было складывать товары только из одинаковых классов продуктов.
- Новая функциональность не дает возможности сложить объекты разных классов, например смартфон и траву газонную. При попытке сложения таких экземпляров выбрасывается ошибка TypeError.
- Для реализации ограничений используется функция type().

Ограничения добавления продукта:
- Доработан метод, который добавляет продукт в категорию таким образом, чтобы не было возможности добавить вместо продукта или его наследников любой другой объект.
- Для реализации ограничений используются функции issubclass() или isinstance().

### ДЗ 14.2

Режимы доступа к атрибутам:
- Атрибут списка товаров класса Category имеет приватный режим доступа.
- Реализован метод add_product() для добавления продукта в атрибут products.
- Метод add_product() принимает на вход аргументы self и продукт для добавления.
- В приватный атрибут products продукт добавляется через метод append.
- Метод add_product() не возвращает значения.
- Метод add_product() в классе Category прибавляет 1 к класс-атрибуту «счетчик продуктов».

Геттер:
- Реализован геттер для приватного атрибута products с использованием декоратора @property.
- Геттер возвращает строку со всеми продуктами в приватном атрибуте products.
- Имя метода-геттера — products.
- Каждый продукт в геттере сформирован как строка строго по шаблону: "Название продукта, X руб. Остаток: X шт.\n".

Класс-метод:
- Реализован класс-метод new_product() с помощью декоратора @classmethod.
- Метод new_product() принимает на вход два аргумента — cls и продукт для создания в виде словаря — и возвращает экземпляр класса на основе данных словаря.

Работа с атрибутом цены:
- Атрибут «цена» класса Product имеет приватный режим доступа.
- Реализован геттер для атрибута «цена» с использованием декоратора @property. Геттер возвращает значение приватного атрибута цены.
 -Название метода-геттера — price.
- Для приватного атрибута «цена» реализован сеттер в классе Product с использованием декоратора @price.setter.
- Сеттер принимает два аргумента — self и новое значение цены.
- В сеттере реализована проверка на положительное значение новой цены.
- Если новое значение цены положительное, то в атрибуте обновляется значение цены.
- Если новое значение цены не положительное, то новое значение приватному атрибуту цены не присваивается и выводится сообщение «Цена не должна быть нулевая или отрицательная» в консоль.
- Сеттер не возвращает никаких значений.
- Название метода-сеттера — price.

### модуль [categories_products.py](src%2Fcategories_products.py):

class Product - Класс для представления товара.
